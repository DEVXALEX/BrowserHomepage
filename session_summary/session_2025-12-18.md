## üìù Key Learnings
-   **Fetch vs Pull**: Fetch is safe (preview), Pull is action (merge).
-   **Rebase**: Use `git pull --rebase` for a linear history when you haven't pushed yet.
-   **Abort**: `git rebase --abort` is the "Undo" button when things go wrong.
-   **Branching**: Always create a branch *before* touching files to avoid "detached head" or dirty working tree states.
-   **Security**: Plain text storage is bad. Encryption with a session-based PIN is a solid "middle ground" for local apps.
-   **Git Log**: `git log -n 5 --oneline` is the best way to quickly check recent history.
